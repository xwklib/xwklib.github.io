容器概念、Docker命令使用

Docker Desktop本质：

  • Windows——通过Docker Client远程连接到Docker Engine
    • Hyper-V
      • Linux
        • Docker Engine

容器

轻量级的虚拟机，共用主机内核，拥有独立的文件系统、网络空间和进程空间视图等。

  • 与虚拟机的区别

    虚拟机是操作系统级别的资源隔离，容器是轻量级的资源隔离

  • Docker的三部分

    • Docker Client（在Windows）

    • Docker Daemon（Docker Engine）（在Linux）

    • Docker registires（Docker仓库）

基本命令

小知识：docker命令参数中若包含两个字母及以上，一般都是–，如--name, --rm， --env-file=XXX其他的就是-，如-d, -p, -v

  • 获取容器的ip地址：cat /etc/hosts

镜像与容器

镜像名：镜像的Tag（类似于版本号）、镜像的ID、镜像的大小

volumes：Docker管理宿主机文件系统的一部分，默认位于/var/lib/dokcer/volumes目录中

容器网络

  • none网络，–net=none

    没有网络（与外界不通）

  • host网络，–net=host

    与宿主机同一个网络

  • bridge网络 *，–net=bridge

    （默认的）

    每创建一个容器，都会创建一个虚拟的网卡，并将端口映射到宿主机中

  • container模式， –net=container:NAME_or_ID
