  • localhost:8888/服务名/prod

    访问的配置文件则为：服务名-prod.yml，然后从中取出一些需要的数据，如果该配置文件中没有需要的数据，则会再次从服务名.yml中找

nc命令是上一节课section11的课后作业所打包的镜像，命令的作用就是返回configserver在$CONFIGSERVER_PORT端口的运行状态；这条语句的作用是：若配置服务没有启动，则一直等待。

在docker-compose.xml中对非配置服务添加depends_on属性可以达到类似的效果，但根本区别在于无法保证配置服务启动成功，只能保证晚于配置服务启动，示例如：licensingservice.depends_on: configserver

若数据库服务没有启动，则一直等待。

配置服务的密码加密

配置

  1. 在服务器上：下载并安装加密所需的Oracle JCE jar（ jce_policy-8）

    • 将local_policy.jar、US_export_policy.jar复制到$JAVA_HOME/jre/lib/security目录

    JCE: Unlimited Strength Java Cryptography Extension

  2. 在docker-compose中的services.configserver.environment配置共享密钥：设置环境变量ENCRYPT_KEY（字符串）

    当加密和解密用的KEY是同一个，则成为对称加密，否则是非对称加密

    对称加密（共享密码）、非对称加密（公钥/私钥）

使用

  1. 原理

    如果发现有ENCRYPT_KEY，则自动添加两个端点：/encrypt和/decrypt

  2. 加密

    对http://localhost:8888/encrypt发送POST请求，并将密码放进body（以raw的形式）

  3. 解密

    对http://localhost:8888/decrypt发送POST请求，并将加密后的密码放进body（以raw的形式）

  4. 加密后的密码添加前缀放入配置文件中
