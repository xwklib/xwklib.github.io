Linux使用yum安装MySQL8.0并设置远程访问（腾讯云服务器）

|阅读量:|评论数:

TMD，折腾了两天，在CSDN上看了好多文章，用了docker安装MySQL，结果发现总是报错

2003 - Can’t connect to MySQL server on IP地址 (10060)
2003 - Can’t connect to MySQL server on IP地址(10038)
不过捣鼓了两天，好在终于成功解决了连接问题，现在就来分享下解决流程。

2003 - Can’t connect to MySQL server on IP地址 (10060)
首先，对于MySQL 10060问题，
可能出现的问题是：

  • bind-address 对于连接的ip做了一个限定
    解决方法：在/etc/my.cnf中找到bind-address并将那一行注释掉

  • 数据库用户只能在指定的ip上连接
    解决方法：登录MySQL，使用如下的一些代码：

    至此就可以了

  • 网络，防火墙可能导致的问题

    解决方法：在本地关闭防火墙（因为我老早之前就关掉了，所以我没遇到过这个问题）

  • 未开放3306端口

    问题很简单，方法自行百度

navicat链接时报 2059 错误

解决方法：alter user 'root'@'%' identified with mysql_native_password by 'yourPassword';

需要说明的是，这个错误我没遇到过，但我还是照做了，这是前后差别^1：

2003 - Can’t connect to MySQL server on IP地址(10038)

这个问题说简单也简单，但是困扰了我很久，也是最想让我骂娘的地方

可能原因如下：

  • Linux上的mysql服务未启动

    解决方法：systemctl start mysqld

  • 3306端口未开放

    我百度了一下，发现有好多是阿里云服务器报10038的错误，说是安全组的问题，我一想，腾讯云服务器也有安全组啊，于是我就照猫画虎地改了腾讯云的安全组，结果发现并没有什么卵用，于是有了下面的解决方法

    腾讯云服务器解决方案

    • 根据个人情况的不同，在Linux防火墙里开放3306端口：
      • 如果你是firewalld（默认防火墙），则执行：
        关于iptables的一些命令 
            
    启动一个服务：systemctl start firewalld.service
    关闭一个服务：systemctlstop firewalld.service
    重启一个服务：systemctlrestart firewalld.service
    显示一个服务的状态：systemctlstatus firewalld.service
    在开机时启用一个服务：systemctlenable firewalld.service
    在开机时禁用一个服务：systemctldisable firewalld.service
    查看服务是否开机启动：systemctlis-enabled firewalld.service
    查看已启动的服务列表：systemctllist-unit-files|grep enabled
    查看启动失败的服务列表：systemctl--failed
    查看版本： firewall-cmd --version
    查看帮助： firewall-cmd --help
    显示状态： firewall-cmd --state
    查看所有打开的端口： firewall-cmd--zone=public --list-ports
    更新防火墙规则： firewall-cmd --reload
    查看区域信息:  firewall-cmd--get-active-zones
    查看指定接口所属区域： firewall-cmd--get-zone-of-interface=eth0
    拒绝所有包：firewall-cmd --panic-on
    取消拒绝状态： firewall-cmd --panic-off
    查看是否拒绝： firewall-cmd --query-panic
    查看
    firewall-cmd --zone=public --query-port=3306/tcp
    删除
    firewall-cmd --zone=public --remove-port=3306/tcp --permanent
      • 如果是iptables，则执行：
        关于firewalld的一些命令 
            
    检查是否安装了iptables 
    service iptables status
    安装iptables 
    yum install -y iptables 
    升级iptables 
    yum update iptables 
    安装iptables-services 
    yum install iptables-services
    systemctl disable iptables #禁止iptables服务 
    systemctl stop iptables #暂停服务 
    systemctl enable iptables #解除禁止iptables 
    systemctl start iptables #开启服务
    • 在服务器的防火墙里添加一条规则，开放3306端口
